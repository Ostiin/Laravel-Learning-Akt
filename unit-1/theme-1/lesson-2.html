Возможности фреймворка Laravel

Консоль Artisan

Artisan — интерфейс командной строки, включенный в Lavarel. Он позволяет генерировать модели, новые тесты, контроллеры, уведомления из командной строки. Это намного удобнее, чем копировать откуда-либо шаблон класса или писать его вручную. Чтобы посмотреть команды, вы можете воспользоваться командой list: 

list: php artisan list

Консоль используют для работы с базами данных, управления контроллером и других действий. Разработчик может написать для нее собственные команды.

Eloquent ORM

ORM — это технология программирования для связи базы данных и языка программирования. Использование ORM позволяет ускорить разработку. Для PHP существует множество реализаций ORM, но Laravel пользуется собственной. Она называется Eloquent и работает по схеме ActiveRecord, согласно которой каждой таблице в базе соответствует один класс. Eloquent любят за удобство и возможность писать понятный код, который легко поддерживать, защиту от SQL-инъекций — способа взлома сайтов и программ. ORM можно загрузить отдельно от остальных компонентов Laravel.

Конструктор Fluent

Это конструктор для быстрого построения запросов к базе, который полностью совместим с ядром Eloquent ORM. Fluent — мощный инструмент, позволяющий с легкостью модифицировать записи в базе данных. Все запросы используют подготовленные выражения и защищены от SQL-инъекций.

Шаблонизатор Blade

Шаблонизаторы используются для превращения HTML-шаблонов в готовые страницы. Шаблоны — это заготовки для будущих веб-страниц, которые включают HTML-верстку без контента и PHP-код. Задача программы-шаблонизатора — выполнить PHP-код и подставить в шаблон контент, чтобы превратить его в готовую страницу. PHP в итоговой веб-странице быть не должно.

Blade — это шаблонизатор фреймворка Laravel. Он не имеет ограничений на чистый PHP в шаблонах — это удобнее для backend-разработчика. Отсутствие ограничений не создает дополнительной нагрузки на приложение.

Валидация

Валидация — проверка входящих данных. Она нужна, чтобы случайная ошибка была вовремя найдена и обработана. Логику валидации можно написать самостоятельно. Если пользователь введет неверные данные, он попадет на предыдущую страницу. Сайт не «упадет» и останется работоспособным.

Система контроля версий баз данных

Это позволяет проводить миграции — модификации структуры базы данных. С помощью фреймворка можно гибко управлять миграцией прямо через Artisan: запускать, откатывать по отдельности или полностью, видоизменять.

Юнит-тестирование

Его еще называют модульным тестированием. Это «базовый уровень» проведения тестов. На работоспособность тестируются отдельные модули кода. Это нужно, чтобы позже не столкнуться с ошибкой на более высоких уровнях.

Этапы тестирования:

В Laravel по умолчанию встроена система PHPUnit — инструмент, который позволяет создавать и проводить тесты. С PHPUnit это проще, чем вручную.

Аутентификация

С Laravel можно реализовать вход на сайт с помощью логина и пароля либо через социальные сети.

Например, пакеты Laravel Passport и Laravel Socialite позволяют входить через API других приложений с помощью стандарта OAuth. Это протокол авторизации, который дает возможность авторизоваться с помощью учетной записи другого сайта, чаще всего соцсети.

Пакет Laravel Sanctum отвечает за аутентификацию для простых приложений и одностраничных сайтов. Он более легковесный, чем Passport, и авторизует пользователя с помощью системы токенов — физических устройств.

Структура каталога, введение

Стандартная структура приложений Laravel предназначена для того, чтобы стать отличной отправной точкой как для больших, так и для малых приложений. Однако вы вольны организовать свое приложение так, как вам нравится. Laravel практически не накладывает ограничений на место расположения того или иного класса - при условии, что Composer может автозагрузить этот класс.

Корневой каталог

Каталог приложений

Каталог app содержит основной код вашего приложения. Мы рассмотрим этот каталог более подробно в ближайшее время, однако почти все классы вашего приложения будут находиться в этом каталоге.

Каталог Bootstrap

Каталог bootstrap содержит файл app.php, который загружает фреймворк. В этой директории также находится каталог cache, который содержит файлы, созданные фреймворком для оптимизации производительности, такие как файлы кэша маршрутов и сервисов. Обычно вам не нужно изменять какие-либо файлы в этой директории.

Каталог Config

Каталог config, как следует из названия, содержит все файлы конфигурации вашего приложения. Очень полезно прочитать все эти файлы и ознакомиться со всеми доступными вам опциями.

Каталог баз данных

Каталог database содержит миграции базы данных, фабрики моделей и семена. При желании вы можете использовать этот каталог для хранения базы данных SQLite.

Каталог Public

В каталоге public находится файл index.php, который является точкой входа для всех запросов, поступающих в ваше приложение, и настраивает автозагрузку. В этой директории также хранятся ваши активы, такие как изображения, JavaScript и CSS.

Каталог Resources

В директории resources содержатся представления, а также сырые, некомпилированные активы, такие как CSS или JavaScript.

Каталог Routes

Каталог routes содержит все определения маршрутов для вашего приложения. По умолчанию в Laravel включено несколько файлов маршрутов: web.php, api.php, console.php и channels.php.

Файл web.php содержит маршруты, которые RouteServiceProvider помещает в группу web middleware, что обеспечивает состояние сессии, защиту CSRF и шифрование cookie. Если ваше приложение не предлагает RESTful API без состояния, то все ваши маршруты, скорее всего, будут определены в файле web.php.

Файл api.php содержит маршруты, которые RouteServiceProvider помещает в группу api middleware. Эти маршруты должны быть статичными, поэтому запросы, поступающие в приложение по этим маршрутам, должны аутентифицироваться с помощью токенов и не будут иметь доступа к состоянию сессии.

Файл console.php - это место, где вы можете определить все ваши консольные команды, основанные на закрытиях. Каждое закрытие привязывается к экземпляру команды, что обеспечивает простой подход к взаимодействию с методами ввода-вывода каждой команды. Хотя этот файл не определяет HTTP-маршруты, он определяет консольные точки входа (маршруты) в ваше приложение.

Файл channels.php - это место, где вы можете зарегистрировать все каналы трансляции событий, которые поддерживает ваше приложение.

Каталог Storage

Каталог storage содержит ваши журналы, скомпилированные шаблоны Blade, сессии на основе файлов, файловые кэши и другие файлы, созданные фреймворком. Эта директория разделена на директории app, framework и logs. Каталог app может использоваться для хранения любых файлов, генерируемых вашим приложением. Каталог framework используется для хранения файлов и кэшей, созданных фреймворком. Наконец, каталог logs содержит файлы журналов вашего приложения.

Директория storage/app/public может использоваться для хранения пользовательских файлов, например аватаров профиля, которые должны быть общедоступны. Вы должны создать символическую ссылку в public/storage, которая будет указывать на эту директорию. Создать ссылку можно с помощью команды php artisan storage:link Artisan.

Каталог тестов

Каталог tests содержит автоматизированные тесты. Примеры юнит-тестов PHPUnit и функциональных тестов предоставляются из коробки. Каждый класс тестов должен иметь суффикс со словом Test. Вы можете запустить свои тесты с помощью команд phpunit или php vendor/bin/phpunit. Или, если вы хотите получить более подробное и красивое представление результатов тестирования, вы можете запустить свои тесты с помощью команды php artisan test.

Каталог Vendor

Каталог vendor содержит ваши зависимости от Composer.

Каталог приложений

Большая часть вашего приложения находится в каталоге app. По умолчанию этот каталог имеет имя App и автозагружается Composer'ом, используя стандарт автозагрузки PSR-4.

Каталог app содержит множество дополнительных каталогов, таких как Console, Http и Providers. Считайте, что каталоги Console и Http предоставляют API к ядру вашего приложения. Протокол HTTP и CLI являются механизмами взаимодействия с вашим приложением, но не содержат логики приложения. Другими словами, это два способа передачи команд вашему приложению. Каталог Console содержит все ваши команды Artisan, а каталог Http - контроллеры, промежуточное ПО и запросы.

По мере использования команд make для создания классов внутри каталога app будет создаваться множество других каталогов. Так, например, каталог app/Jobs не будет существовать до тех пор, пока вы не выполните команду make:job для создания класса job.

Каталог Broadcasting

Каталог Broadcasting содержит все классы каналов вещания для вашего приложения. Эти классы создаются с помощью команды make:channel. По умолчанию этот каталог не существует, но он будет создан при создании первого канала.

Каталог Console

Каталог Console содержит все пользовательские команды Artisan для вашего приложения. Эти команды могут быть созданы с помощью make:command. В этом каталоге также находится ядро консоли, в котором регистрируются пользовательские команды Artisan и определяются запланированные задачи.

Каталог Events

По умолчанию этот каталог не существует, но он будет создан командами event:generate и make:event. В каталоге Events содержатся классы событий. События могут использоваться для оповещения других частей вашего приложения о том, что произошло то или иное действие, что обеспечивает большую гибкость и развязку.

Каталог Exceptions

Каталог Exceptions содержит обработчик исключений вашего приложения, а также является хорошим местом для размещения любых исключений, вызываемых вашим приложением. Если вы хотите изменить способ регистрации или отображения исключений, вам следует модифицировать класс Handler в этом каталоге.

Каталог Http

Каталог Http содержит контроллеры, промежуточное ПО и запросы к формам. Почти вся логика обработки запросов, поступающих в ваше приложение, будет размещена в этом каталоге.

Каталог Jobs

По умолчанию этот каталог не существует, но он будет создан, если вы выполните команду make:job. В директории Jobs хранятся задания, которые можно поставить в очередь для вашего приложения. Задания могут быть поставлены в очередь вашим приложением или выполняться синхронно в течение текущего жизненного цикла запроса. Задания, которые выполняются синхронно во время текущего запроса, иногда называют "командами", поскольку они являются реализацией командного шаблона.

Каталог Listeners

По умолчанию этот каталог не существует, но он будет создан, если вы выполните команды event:generate или make:listener. Каталог Listeners содержит классы, которые обрабатывают ваши события. Слушатели событий получают экземпляр события и выполняют логику в ответ на его срабатывание. Например, событие UserRegistered может быть обработано слушателем SendWelcomeEmail

Каталог Mail

По умолчанию этот каталог не существует, но он будет создан, если вы выполните команду make:mail. Каталог Mail содержит все ваши классы, которые представляют электронные письма, отправляемые вашим приложением. Объекты Mail позволяют инкапсулировать всю логику создания письма в одном простом классе, который можно отправить с помощью метода Mail::send.

Каталог Models

Каталог Models содержит все ваши классы моделей Eloquent. Eloquent ORM, входящий в состав Laravel, предоставляет красивую и простую реализацию ActiveRecord для работы с базой данных. Каждая таблица базы данных имеет соответствующую "Модель", которая используется для взаимодействия с этой таблицей. Модели позволяют вам запрашивать данные в таблицах, а также вставлять новые записи в таблицу.

Каталог уведомлений

По умолчанию этот каталог не существует, но он будет создан, если вы выполните команду make:notification. Каталог Notifications содержит все "транзакционные" уведомления, которые отправляются вашим приложением, например, простые уведомления о событиях, происходящих внутри вашего приложения. Функция уведомлений Laravel абстрагирует отправку уведомлений через различные драйверы, такие как электронная почта, Slack, SMS или хранение в базе данных.

Каталог Policies

По умолчанию этот каталог не существует, но он будет создан, если вы выполните команду make:policy. Каталог Policies содержит классы политик авторизации для вашего приложения. Политики используются для определения того, может ли пользователь выполнить то или иное действие над ресурсом.

Каталог Providers

Каталог Providers содержит все поставщики услуг для вашего приложения. Провайдеры загружают ваше приложение, связывая сервисы в контейнере сервисов, регистрируя события или выполняя любые другие задачи, чтобы подготовить приложение к входящим запросам.

В новом приложении Laravel эта директория уже будет содержать несколько провайдеров. При необходимости вы можете добавить в эту директорию своих провайдеров.

Каталог Rules

По умолчанию этот каталог не существует, но он будет создан, если вы выполните команду make:rule. Каталог Rules содержит объекты пользовательских правил валидации для вашего приложения. Правила используются для инкапсуляции сложной логики проверки в простой объект.
