Создание контроллеров

Для чего нужны контроллеры

Контроллеры в Laravel используются для обработки запросов и управления логикой приложения. Они играют важную роль в архитектуре MVC (Model-View-Controller) фреймворка Laravel.

Контроллеры в Laravel помогают разделить логику приложения на отдельные компоненты, что обеспечивает более чистый и организованный код. Они принимают запросы от пользователей, взаимодействуют с моделями и представлениями, и возвращают результаты обратно пользователю.

Создание стандартного контроллера

Чтобы быстро сгенерировать новый контроллер, можно выполнить команду make:controller Artisan. По умолчанию все контроллеры для вашего приложения хранятся в каталоге app/Http/Controllers: 

php artisan make:controller UserController

Теперь можно создать метод, который будет вовзращать сообщение "Тест Контроллера", код будет выглядеть следующим образом: 

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    public function index() {
        return 'Тест Контроллера';
    }
}

В данном случае index - метод контроллера, который возвращает наш текст.

Вызов контроллера по маршруту

Для того чтобы вызвать контроллер по маршруту, нужно создать этот маршрут. Переходим в уже знакомый нам файл web.php, и для начала подключаем наш контроллер с помощью данного кода: 

use App\Http\Controllers\UserController;

После этого мы добавляем новый маршрут: 

Route::get('/test', [UserController::class, 'index']);

В данном коде /test - маршрут, UserController::class - определение контроллера, а index - вызов метода у контроллера. В целом код маршрутов выглядит следующим образом: 

<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\UserController;

Route::get('/', function () {
    return view('welcome');
});

Route::get('/hello', function () {
    return 'Hello World';
});

Route::get('/user/{id}', function (string $id) {
    return 'User '.$id;
});

Route::get('/test', [UserController::class, 'index']);
