Основы маршрутизации

Маршруты определяют что именно фреймворк будет показывать при обращении к определённому адресу. К примеру базовое использование маршрутов: 

use Illuminate\Support\Facades\Route;
Route::get('/hello', function () {
    return 'Hello World';
});

В этом примере при нахождении в адресной строке "/greeting", выводится страница с надписью "Hello World". Давайте посмотрим как это выглядит.

Для начала нам нужно открыть папку нашего проекта, который мы создали в прошлом уроке, в Visual Studio Code. Уже там нужно открыть файл web.php, который находится в папке routes. Выглядит это примерно так:

Теперь вставим код выше, для того чтобы посмотреть работу кастомного маршрута.

Для того чтобы проверить работу маршрута нужно запустить через Open Server наш проект, и добавить в адресную строку сам маршрут, в нашем случае это /hello. Отлично! Мы види "Hello World".

Доступные методы маршрутизации

Зачастую маршрутизация происходит с помощью get метода, но также есть и следующие методы:

Route::get($uri, $callback);
Route::post($uri, $callback);
Route::put($uri, $callback);
Route::patch($uri, $callback);
Route::delete($uri, $callback);
Route::options($uri, $callback);

Встраивание маршрутов в HTML

Если вы хотите встраивать маршруты в HTML код, помните что для методов POST, PUT, PATCH, или DELETE нужно указывать @csrf в коде. В противном случае запрос будет отклонен. 

<form method="POST" action="/profile">
    @csrf
    ...
</form>

Редирект маршрутов

В Laravel есть возможность редиректа маршрутов. К пример, в коде ниже при переходе по маршруту /here вас отправит не маршрут /there. 

Route::redirect('/here', '/there');

Передача данных через маршруты

С помощью маршрутов можно предавать данные, к примеру в коде ниже, мы передаем в маршруте $id, в зависимости от введенного id в адресную строку и будет вывод результата. 

Route::get('/user/{id}', function (string $id) {
    return 'User '.$id;
});

К примеру если ввести /users/1, будет выведено 1.

Опциональные параметры

Иногда вам может потребоваться указать параметр маршрута, который не всегда присутствует в URI. Это можно сделать, поставив знак ? после имени параметра. Убедитесь, что соответствующая переменная маршрута имеет значение по умолчанию: 

Route::get('/user/{name?}', function (?string $name = null) {
    return $name;
});
    
Route::get('/user/{name?}', function (?string $name = 'John') {
    return $name;
});

В первом маршруте, если не указать имя мы получим вывод "null", а во втором уже "John".

Регулярные выражения в маршрутах

Вы можете ограничить формат параметров маршрута с помощью метода where для экземпляра маршрута. Метод where принимает имя параметра и регулярное выражение, определяющее, как параметр должен быть ограничен: 

Route::get('/user/{name}', function (string $name) {
    // ...
})->where('name', '[A-Za-z]+');
    
Route::get('/user/{id}', function (string $id) {
    // ...
})->where('id', '[0-9]+');

Также в Laravel есть готовые шаблоны регулярных выражений.

Имена для маршрутов

Именованные маршруты позволяют удобно генерировать URL-адреса или перенаправления для определенных маршрутов. Вы можете задать имя маршрута, подключив метод name к определению маршрута: 

Route::get('/user/profile', function () {
    // ...
})->name('profile');

Префиксы у маршрутов

Метод prefix можно использовать для префиксации каждого маршрута в группе с заданным URI. Например, вы можете присвоить всем URI маршрутов в группе префикс admin: 

Route::prefix('admin')->group(function () {
    Route::get('/users', function () {
        // Matches The "/admin/users" URL
    });
});
