Применение middleware.

Для чего нужен middleware

Middleware (посредники) в Laravel являются фильтрами для обработки HTTP-запросов. Они позволяют выполнять различные задачи, такие как проверка аутентификации пользователя. В случае, если пользователь не залогинен, middleware перенаправляет его на страницу логина. Если же пользователь уже залогинен, middleware не вмешивается в обработку запроса и передает его дальше по цепочке middleware-посредников к самому приложению.

Однако, проверка авторизации - это не единственная задача, которую могут выполнять middlewares. Они также могут добавлять специальные заголовки (например, CORS http-ответ вашего приложения) или выполнять логирование всех HTTP-запросов.

Создание стандартного middleware

Чтобы создать новый middleware, воспользуйтесь командой make:middleware Artisan: 

php artisan make:middleware EnsureTokenIsValid

Эта команда поместит новый класс EnsureTokenIsValid в каталог app/Http/Middleware. В этом промежуточном ПО мы будем разрешать доступ к маршруту только в том случае, если введенный токен соответствует заданному значению. В противном случае мы будем перенаправлять пользователей обратно на домашний URI: 

<?php

namespace App\Http\Middleware;
    
use Closure;
use Illuminate\Http\Request;
use Symfony\Component\HttpFoundation\Response;
    
class EnsureTokenIsValid
{
    /**
        * Handle an incoming request.
        *
        * @param  \Closure(\Illuminate\Http\Request): (\Symfony\Component\HttpFoundation\Response)  $next
        */
    public function handle(Request $request, Closure $next): Response
    {
        if ($request->input('token') !== 'my-secret-token') {
            return redirect('home');
        }
    
        return $next($request);
    }
}

На данном этапе это все, что нужно знать о middleware.
