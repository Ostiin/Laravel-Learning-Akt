Настройка БД

Чтобы подключить БД, нужно перейти в файл .env и изменить следущие параметры: 

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=mydb
DB_USERNAME=root
DB_PASSWORD=

Также с помощью phpMyAdmin нужно создать БД с именеи "mydb".

Настройка миграций

Изначально, чтобы миграции не выдавали ошибки, нужно изменить содержимое app/Providers/AppServiceProvider.php на следущее: 

<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Illuminate\Support\Facades\Schema;

class AppServiceProvider extends ServiceProvider
{
    /**
        * Register any application services.
        */
    public function register(): void
    {
        //
    }

    /**
        * Bootstrap any application services.
        */
    public function boot(): void
    {
        Schema::defaultStringLength(191);
    }
}

Laravel имеет некоторые миграции по умолчанию. Они нам не нужны, поэтому удалим все миграции из папок database/migrations и vendor/laravel/sanctum/database/migrations.

Создание миграции

После мы создаем миграцию с помощью команды: 

php artisan make:migration create_users_table

Переходим в database/migrations и открываем нашу миграцию. Меням содержимое миграции на следущее: 

<?php

use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration
{
    /**
        * Run the migrations.
        */
    public function up(): void
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->string('password');
            $table->timestamps();
        });
    }

    /**
        * Reverse the migrations.
        */
    public function down(): void
    {
        Schema::dropIfExists('users');
    }
};

В данном коде мы определяем поля таблицы, которые будут создаваться при миграции.

Отлично! Теперь можно мигрировать нашу таблицу с помощью команды: 

php artisan migrate

Если вы сделали все верно, то в БД mydb появятся 2 таблицы. Ту, которую мы мигрировали, и таблица со всеми миграциями от самого Laravel.

Создание модели

Теперь для взаимодействия с БД нужна модель. Для начала нужно удалить все модели из папки app/Models. После создаем свою модель, чтобы ее создать: 

php artisan make:model User

Модель создана и сохранена в папке app/Models. Измените код модели на следущий: 

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;

class User extends Model
{
    use HasFactory;

    protected $fillable = [
        'name', 'password'
        ];
}

В данной модели мы просто объявляем поля, которые есть в нашей таблице.
