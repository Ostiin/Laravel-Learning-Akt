Настройка валидации формы

Создание формы

Для того чтобы провести валидацию формы, нужно ее создать. Наша форма будет находится в файле resources/views/validate.blade.php. Файл долежно содержать следующий код: 

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Форма валидации</title>
</head>
<body>
    <form action="/validate/show">
        <label for="title">Заголовок: </label><input type="text" id="title" name="title">
        <label for="body">Основной контент: </label><input type="text" id="body" name="body">
        <input type="submit">
    </form>
    @if ($errors->any())
        <div class="alert alert-danger">
            <ul>
                @foreach ($errors->all() as $error)
                    <li>{{ $error }}</li>
                @endforeach
            </ul>
        </div>
    @endif
</body>
</html>

В данном представлении есть форма, которая отправляет данные по маршруту /validate/show. Также после формы идет вывод ошибок валидации. На данном этапе он не работает, но как только мы пропишем логику валидации, все заработает.

Настройка контроллера

Следующим шагом будет создание контроллера, который и будет обрабатывать валидацию. У нас уже был создан контроллеры в прошлых уроках, поэтому перейдем в него по следующему пути: app/Http/Controllers/UserController.php и вставим туда следующий код: 

<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class UserController extends Controller
{
    public function index() {
        return 'Тест Контроллера';
    }

    public function show(Request $request)
    {
        $validated = $request->validate([
            'title' => 'required|max:25',
            'body' => 'required',
        ]);
        return view('validate-res');
    }
}

В этом контроллере мы добавили метод show, который принимает данные из формы по их name, и валидирует их с помощью специальных указателей. В нашем случае required значит то, что поле обязательное, а max:25 ограничивает длину строки до 25 символов.

Если валидация не обнаружила ошибок, просходит возвращение представления validate-res, которое нам нужно создать. Для этого создаем файл resources/views/validate-res.blade.php, а также добавляем в него следущий код: 

Заголовок: {{ $_REQUEST['title'] }} <br><br>
Тело: {{ $_REQUEST['body'] }}

В этом представлении мы выводим данные, если они прошли валидацию.

Создание маршрутов

Осталось добавить маршруты, по которым мы попадем на соответствующие страницы, для этого перейдем в файл web.php и добавим туда следующие маршруты: 

Route::get('/validate', function () {
    return view('validate');
});

Route::get('/validate/show', [UserController::class, 'show']);

В коде выше были созданы два маршрута. Маршрут /validate ведет на страницу с формой, а /validate/show в свою очередь отсылает на контроллер с методом show.
