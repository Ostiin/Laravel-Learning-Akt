Для чего нужны представления

Views (представления, отображения, шаблоны) обычно содержат HTML-код вашего приложения и представляют собой удобный способ разделения бизнес-логики и логики отображения информации. Шаблоны хранятся в папке resources/views.

Создание представлений

Вы можете создать представление, поместив файл с расширением .blade.php в каталог resources/views вашего приложения, или с помощью команды: 

php artisan make:view first-view

После создание представления через консоль, оно появится в папке resources/views/first-view.blade.php, и будет иметь следующее содержимое:

Работа с представлениями

Обращение к представлению в маршруте

Первым делом необходимо в наш существующий шаблон добавить какой-то текст, который будет выводиться при переходе по маршруту. В нашем случае это будет следующая конструкция: 

<div>
    Привет мир! Это вывод содержимого шаблона.
</div>

Теперь переходим в routes/web.php и добавляем туда следующий маршрут: 

Route::get('/first-view', function () {
    return view('first-view');
});

Теперь при переходе по маршруту /first-view выводится следующая страница:

Использование переменных в представлении

Для того чтобы отобразить переменную в представлении, нужно для начала ее передать. Это можно сделать с помощью маршрута, для этого изменим предыдущий маршрут на следующий: 

Route::get('/first-view', function () {
    return view('first-view', ['name' => 'Maks']);
});

Также нужно изменить файл first-view.blade.php вставив следующий код: 

<div>
    {{ $name }}
</div>

Конструкция {{ name }} отвечает за вывод переменной, переданной с помощью маршрута. Результат будет следующий:

Наследование представлений

Представления также могут наследоваться. Для начала создадим файл second-view.blade.php в папке resources/views.

В данный файл вставляем следующий код: 

@extends('first-view')

В данном случае @extends('first-view') выполняет наследование шаблона second-view.blade.php.

Осталось добавить следующий маршрут: 

Route::get('/second-view', function () {
    return view('second-view', ['name' => 'Maks']);
});

Теперь при переходе по маршруту /second-view будет выводить следующее:
